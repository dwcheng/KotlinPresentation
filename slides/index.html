What is Kotlin?

  • Statically typed JVM language
  • Java like syntax
  • Addresses many common complaints about Java
First off, what is Kotlin? Kotlin is a statically typed, JVM language which was designed to be like Java, but address many of the common complaints people have about it Hopefully by the end of this presentation I'll have convinced you to use Kotlin in your next project

Why?

  • Java Interop
  • Modern Language Features
    • Null Safety
    • Concise Syntax
  • First class language for Android
  • Growing popularity

First is low barrier to entry. I'll go into more detail later, but basically Kotlin allows you to reuse existing Java code and libraries

Kotlin provides developers with many shiny tools to get the job done

I consider this to be one of Kotlin's best features. Basically if you're writing idiomatic Kotlin, you will never run into a NPE

Lastly I'll talk about some language features that make reading and writing Kotlin nice

Interop

Using Kotlin means you can keep using existing libraries and code

One of the big pain points when starting with a new language is rewriting code

This also means that you can do an incremental rewrite of your codebase rather than starting from scratch

Null Safety

In Kotlin, null-pointers are handled at compile time rather than runtime

If the compiler detects a possible NPE, it will not compile
